# 公式LINE ステップ配信シナリオ（AIオフィス集客）＋設計の考え方

> 集客用 公式LINE「AIオフィス｜LINEで動くAIチーム」(@453blnmx) の配信設計。
> X（集客）→ 公式LINE（リスト化・教育・販売）→ 無料相談 → Stripe決済 の“教育〜販売”を担う。

---

## 設計の考え方（忘れない・全配信の土台）

### リストマーケの4ステップ／LINEの守備範囲
1. 認知・集客（X等が得意。LINEは弱い→「友だち追加」導線）
2. リスト化（友だち追加＝ここからLINEの本領）
3. 教育（ステップ配信で価値・必要性を段階的に）
4. 販売／行動促進（CTAで相談・申込へ）
→ **LINEが担うのは「リスト化以降（特に教育と販売）」**。集客はXで。

### 3つのシナリオ（全体の役割）
- **教育シナリオ**：見込み客に「良さそう」を作る。**メリットを積み上げる（ポジティブ方向）**。
- **セールスシナリオ**：迷い・不安を消し行動させる。**「やらない理由」を潰す（ネガティブを消す方向）**。
- アフターフォロー：購入後のリピート/紹介（今回はスコープ外）。
→ 教育で欲しい気持ち、セールスで不安解消、の両輪。**教育を省くと比較検討層は離脱**。

### 各メッセージに必ず3点セット
- **目的**：この1通で何を理解・感じ・行動してほしいか
- **CTA**：次の具体行動。**小学生でも分かる直球**（「『デモ』と送ってください」）。遠回しは離脱。
- **ユーザー心理**：受け取る瞬間の気持ち。**登録からの経過時間**で変わる（直後＝熱量MAX／数日＝登録理由は覚えてる／1週間＝冷め気味、不安を言語化して消す）。

### 初回オファーを“先”に出す
教育を全部終えてからオファー、ではなく、**登録直後（あいさつ）に初回オファー（デモ視聴）**を置く。
→ 顕在層を取りこぼさず、後続の教育が「見るか決めるための情報」として読まれ開封率が上がる。

## このシナリオの流れ（教育→セールス）
```
登録直後：あいさつ＝初回オファー（デモを見る判断軸）
 →翌日   教育①：何ができるか（守備範囲）
 →2日後  教育②：外注/採用との比較（自分ごと化）
 →2日後  教育③：不安の先回り（機材いらず・難しくない）
 →2日後  セールス①：料金を透明に＋先着（行動の理由）
 →翌日   セールス②：迷いを消す＋直球CTA（リマインド）
```
設定：0番＝「あいさつメッセージ」、1〜5番＝「ステップ配信」（開始トリガー＝友だち追加）。

## LINEでの設定方法
- **0番（あいさつ）**：ホーム →「あいさつメッセージ」に貼る（友だち追加直後に自動送信）。
- **1〜5番**：左メニュー **「ステップ配信」** → 新規作成 → 開始トリガー **「友だち追加」** → 各メッセージと**待機時間（翌日／2日後…）**を順に追加。
- **応答設定＝チャット（手動）**にして、「デモ/試算/相談」の返信に個別対応。
- 「相談」が来たら→ ヒアリング → 提案 → Stripeの支払いリンクを案内（成約）。
