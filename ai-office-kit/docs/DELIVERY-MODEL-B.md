# 納品モデルB＋(あ) 運用プレイブック

> 決定（2026-06-28）：販売モデルは **B（あなたが管理する1台に集約・運用代行）＋(あ)（顧客ごとに顧客名義のClaude契約）**。
> 理由：顧客のMacに毎回構築するのは非技術者には事故リスクが高い。1台に集約すれば環境が一定で安定。
> 顧客のハードルも下がり売りやすい。Claudeは顧客名義にして容量問題とコストを切り分ける。

---

## 全体像（マンション大家モデル）
- あなた＝大家。**常時起動の1台（Mac mini推奨）＝マンション**を持つ。
- その中に**顧客ごとの“個室”（AIオフィス）**を作る。各個室は**顧客自身のClaude契約**で動く（＝部屋ごとに電気メーターが別）。
- 顧客は自分のLINEで使うだけ。構築・運用・サポートはあなた。
- 収益＝初期構築費（少なめ）＋**月額（家賃）**。Claude実費は顧客直払い。

## 必要なもの（誰が用意するか）
| もの | 担当 | 補足 |
|---|---|---|
| 常時起動Mac 1台 | あなた | Mac mini定番。スリープ無効・自動再起動・電源つけっぱ |
| 安定ネット（有線推奨） | あなた | 落ちると全顧客停止のため重視 |

| LINE公式アカウント＋チャンネル | 顧客名義（設定はあなたが代行） | トークン取得して連携 |

## 顧客ごとの個室の分け方
- Mac1台の中に**顧客ごとのログインユーザー（部屋）を分ける**（各ユーザーが独自の Claude ログインを持てる）。
- 個室ごとに `OFFICE_HOME` と **PORT を分ける**（例：顧客A=18789／顧客B=18790）＝衝突回避。
- 個室ごとに `office.conf` の `MEMBERS`（業種に合わせた社員構成）を設定。
- push運用（既定）で待機消費を抑える。
- ※キットは OFFICE_HOME / PORT / MEMBERS が分離設計なので、1台での複数オフィス運用に対応可能。

## オンボーディング（最初の1件の流れ）
1. ヒアリング（業種・やりたいこと・必要な社員）
2. 顧客に **Claude契約**してもらう（画面共有でサポート）
3. 顧客の **LINE公式アカウント＋チャンネル作成**（画面共有 or 委任）→ トークン取得
4. あなたのMacに **顧客の個室を作成** ＋ 顧客のClaudeでログイン ＋ `install.sh` で構築
5. LINEのWebhookを **あなたのCloudflare** に向ける
6. **受け入れテスト**（LINEで動くか end-to-end）
7. **使い方レクチャー**（顧客はLINEで指示するだけ）
8. 運用開始 → **月額スタート**
