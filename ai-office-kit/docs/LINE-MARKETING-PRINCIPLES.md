## 1. ファネル全体像（LINEが担うのは「リスト化以降」）
```
X（認知・集客）→ 友だち追加（リスト化）→ 公式LINE で 教育→販売 → 無料相談 → Stripe決済
```
- 集客はX（LINEは集客が弱い）。**友だち追加＝ここからLINEの本領**。
- LINE内の4部品が役割分担して導線を作る：
  1. **あいさつメッセージ**（即時）：第一印象＋初回オファー（超軽いハードル＝「デモを見る」）。
  2. **ステップ配信**（登録起点・時刻指定）：計画的な教育→セールス。全員に段階配信。
  3. **応答メッセージ**（キーワード）：能動的に動いた反応者だけに即時返信。ステップ配信の受け皿。
  4. **リッチメニュー**（常設）：配信で流れて取りこぼす「あとから来る行動」を回収。
- 最後に**通しテスト**で1本に繋がっているか検証してから公開。

## 2. ステップ配信の原則
- **時刻指定配信**（登録N日後の19:00等）。経過時間指定は深夜配信→ブロックの元。平日17〜20時が当たり。
- **1メッセージ＝1CTA**。同日連投しない（LINEは即日開封が多い＝連投は情報過多）。
- 各通に **目的／CTA／ユーザー心理** の3点セット。CTAは小学生でも分かる直球。
- **初回オファーを先出し**（あいさつでデモ視聴を置く）。顕在層を取りこぼさない。
- 教育（メリット積み上げ・ポジティブ）とセールス（やらない理由を潰す・ネガ消し）の両輪。
- 待機は最短1日・最大30日・日単位（当日配信不可）。※LINE仕様。

## 3. 応答メッセージ（キーワード）の原則
- **役割**：ステップ配信＝登録イベント起点で全員に。応答メッセージ＝ユーザーが能動送信した瞬間に反応者だけへ即時返信。
- **なぜ効く**：能動的にキーワードを送った瞬間が最も熱い。秒で返ると信頼が一段上がる。人手待ちは熱が冷める。
- **補完関係**：配信が「◯◯と送って」と能動行動を促す → 応答メッセージが受け止め、次の行動へ繋ぐ（無音にしない）。
- **キーワードは短く**（2〜3文字）・覚えやすく・コピペしやすく。入力ミスを避ける。
  （例外：リッチメッセージのタップで自動送信されるキーワードは長くてよい＝ユーザーは打たない）
- **一致は原則「部分一致」**（「相談です」等の言い換えも拾う）。完全一致は取りこぼす。
- **判定は新しい登録から順**。一致した時点で停止。キーワードが重ならないよう設計する。

## 4. リッチメニューの原則
- **存在理由**：配信はトーク画面を流れる→あとから情報を取りに戻るのは高ハードル。常設メニューがそれを回収。
- 常設でクリック率が高い。トーク画面の下半分を占有し、開くたび露出する構造的優位。
- **右下ホットゾーン**：右手持ちの親指が最も届く＝右下が押されやすい。**一番押してほしい行動を右下（or 上段大）に**。
- **2層構造**：A＝メインCTA（大領域・常設の入口）／B・C・D＝サブ導線（相談・コンテンツ・信頼補強）。
- **詰め込まない**：選択肢が多いと選べず誰も押さない。「1画面の選択肢も少ないほど反応が上がる」＝1CTAと同じ原則。4分割が目安。
- **遷移先の使い分け**：LINE内で完結する情報は**テキスト送信→応答メッセージ**で（離脱させない）。詳細ページだけ**外部リンク**。
- 画像仕様：大 1200×810px（1+3分割）／小 1200×405px。テキストアクションのキーワードは応答メッセージ側と**完全一致**させる（1文字でもズレると無音）。

## 5. 通しテストの原則（公開前の最後の砦）
- **単体OK≠統合OK**。友だち追加→あいさつ→配信→キーワード→リッチメニューを、1人のユーザーとして通しで体験して検証。
- ステップ配信は**「テスト配信」機能**で各通を即時確認（本番の待機に触れず・配信通数にカウントされない）。配信タイミング自体は設定画面で目視。
- テスト前に**ブロック→削除→再追加**でまっさらにする。
- **ブロックは取り戻せない**（以降メッセージが届かない・広告で取り戻せない固定損失）。だから公開前に必ず通す。
- 機能だけでなく**違和感**も見る：文体の一貫性／画像トーン／つなぎ目の自然さ／情報の重複／CTAの明瞭さ。
- **修正→再テストをセットでループ**（直したつもりで新バグを生むのを防ぐ）。

## 6. タツ版への適用（この方針での実装）
- **キーワード（当面）**：
  - `デモ` … あいさつ画像タップ/本文CTAで送信 → 実演案内（今は手動／後で動画URL）。
  - `あとで` … 2通目2択の「今は忙しい」→ やわらかく後日再案内（学習元の「今忙しいから後で」相当）。
  - `感想` … デモ後に気になった点を送ってもらう受け皿 → お礼＋「相談」案内（学習元の「感想」相当）。
  - `相談` … ヒアリング（業種／任せたい作業／不安）→ 提案 → **自社Stripe**支払いリンク（学習元の「企画→決済」相当。ただし収益断定なし・自社導線）。
- **リッチメニュー（v2で作る）4分割案**：A（上段大）＝**無料相談**（メインCTA）／B＝**デモを見る**（「デモ」テキスト送信）／C＝**サービス紹介**（noteやLP・用意でき次第）／D＝**運営者情報**（自己紹介・実績）。全て自社の遷移先。
- **収益断定は使わない**。感想フォームや決済は自前を用意して実URLを入れる（学習用ダミーは入れない）。
- 現状：テキスト版で稼働中。デモ動画・リッチメッセージ画像・リッチメニュー画像は順次作成して差し替え。
