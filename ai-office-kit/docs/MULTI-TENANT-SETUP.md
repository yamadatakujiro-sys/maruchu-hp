# 1台に複数顧客を載せる手順（個室の分け方）

> モデルB＋(あ)で、あなたの1台に顧客ごとの“個室”を作るための手順と注意。
> ⚠️ 大前提：**1人目はシンプルでよい**。複雑な分離は「2人目を同時に動かす段階」で必要になる。
> 無理に最初から作り込まないこと。

---

## 段階0：1人目（まずはこれだけ）
今お使いのMac（Mac miniは後でOK）で、最小構成で始める。
1. Macを**電源つけっぱ・スリープ無効・自動再起動ON**にする。
2. お客さんの **Claudeアカウントでログイン**（`claude` 初回ログイン）。
3. line-harness＋キットを用意し、`config/office.conf` を設定：
   - `OFFICE_HOME`（このお客さん用フォルダ）／`PORT=18789`／`MEMBERS`（業種別）／`MODE="push"`
4. `./install.sh` で構築 → Cloudflareの土管をこの bridge ポートへ → LINE webhook をその URL に。
5. テスト→レクチャー→運用開始。
→ この段階では「分離」を気にしなくてよい（1顧客＝1Claude＝1オフィス）。

## 段階1：2人目以降（同時稼働させるとき）
顧客ごとに **Claude契約が違う**ため、Claudeログインを分ける必要がある。
クリーンなのは **macOSの「ユーザーアカウント」を顧客ごとに分ける**方法。

顧客1人ぶんの追加手順：
1. システム設定 ＞ ユーザーとグループ で**新規ユーザー**を作成（例：`client-salon`）。
2. そのユーザーでログインし、**そのお客さんのClaudeでログイン**（ユーザーごとに `~/.claude` が独立＝枠が混ざらない）。
3. そのユーザーのホームに line-harness＋キットを用意。
4. `office.conf`：`OFFICE_HOME`＝そのユーザー配下／**`PORT` は顧客ごとに固有**（18789, 18790, 18791…）／`MEMBERS`／`MODE="push"`。
5. `./install.sh`（user単位の launchd に常駐＝顧客ごとに独立）。
6. Cloudflare で**顧客ごとに公開URLを分け**、それぞれの bridge ポートへ。
7. LINE webhook を各顧客の URL に。

### 分離のキモ（混ざらないポイント）
- **Claudeログイン**：macOSユーザーを分ける＝`~/.claude` が別＝顧客の枠で動く。
- **常駐**：ユーザーごとの LaunchAgents で独立。
- **PORT**：必ず顧客ごとに変える（衝突防止）。
- **公開URL**：Cloudflareで顧客ごとに分ける。

## ⚠️ 正直な注意（macOSの制約）
- macOSのユーザー常駐（LaunchAgents）は基本「そのユーザーがログイン中」に動く。**複数ユーザーの常駐を同時に動かし続ける**には、自動ログイン＋常駐の有効化など追加設定が要る（ヘッドレス運用は手間）。
- ＝**(あ)＋macOSユーザー分けは「数人まで」なら現実的**。それ以上の規模になったら下記へ。

## 段階2：規模が増えたら（API方式 (い) へ）
- 顧客が増える／同時稼働が重くなると、ログイン分離より **(い) API方式**が本筋。
- 各オフィスがその顧客のAPIキーで動く形に**キットを改修**（`claude` CLI 依存→API呼び出し）。ログイン分離不要・多顧客に強い。
- これは技術改修なので、必要になった時点で実装する（要相談）。

## まとめ
- **今**：段階0で1人目を回す（悩まない）。
- **2人目同時**：段階1（macOSユーザー分け＋PORT分け）。
- **大規模**：段階2（API方式へ改修）。
