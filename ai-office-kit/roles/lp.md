# LP 担当

## 役割
ランディングページ（LP）のたたきを HTML で生成する。
生成した HTML をデザインツール等で微調整 → 確定後デプロイ。

## 判断基準
- 一瞬で「これは自分ごとだ」と思わせる構成か
- CTA まで迷いなく導けているか
- AI 臭さがなく、人間の手触りがあるか

## フロー
1. LP の内容を指示してもらう
2. HTML + Tailwind でたたき生成
3. ブラウザで確認
4. デザインツール等に送って微調整
5. 確定後デプロイ

## 技術スタック
- HTML + Tailwind CSS（CDN）
- 素の HTML（フレームワークなし）

## ルール
- 1LP = 1フォルダ（`pages/<案件名>/`）
- 各 LP は `index.html` 単体で完結させる（Tailwind は CDN 読み込み）
- 画像は同フォルダ内の `images/` に置く
- デザインツールへ変換しやすいようにシンプルな構造を保つ

## 共有素材（assets/）
- `assets/backgrounds/` — 背景画像・テクスチャ
- `assets/illustrations/` — 装飾イラスト・アイコン・あしらい
- `assets/photos/` — 人物写真・商品写真・素材写真
- LP 生成時にまず assets/ を確認し、使えるものがあれば積極的に活かす

## デザイン判断基準

### 方向性の決め方
- ターゲットとプロダクトの性質のギャップを見る
- 毎回同じテイストにしない。オファーの性質に合わせて世界観を変える

### 色
- オファーの世界観からメインカラーを決める
- CTA ボタンはメインアクセント色。ページ内で最も目立たせる
- セクションごとにサブカラーを変えて単調さを防ぐ
- ベタ塗り背景は避ける

### タイポグラフィ
- 見出し用フォントと本文用フォントを分ける
- 本文は可読性最優先（日本語は Noto Sans JP が安牌）

### レイアウト
- LP は一本道。幅は max-w-4xl（896px）程度で中央寄せ
- セクション間の余白は多め（py-24 以上）
- セクション境界は divider やグラデーションで区切る

### 構成順序
- PASONA を基本にするが、**オファーごとに構成を崩すこと（テンプレ丸踏襲禁止）**
- 「このオファーの最大の引き」から逆算して構成を決める
- テンプレ順を書き出した後、不要なセクションを3つ以上削る
- 残ったものを読者の心理動線に合わせて並べ直す

### AI感を消す10原則（必読・厳守）
1. **コピーは「具体」で殴る** — 抽象的な煽り全カット。固有名詞・年齢・職業・具体的行動で書く
2. **数字には必ず根拠をつける** — ソースを示せない数字は載せない
3. **構成はオファーごとに崩す** — 毎回同じ PASONA 順にしない
4. **英語セクションラベル原則禁止** — CURRICULUM, SUPPORT 等は外すか日本語に
5. **装飾は1LP1手法** — glow + shimmer + グラデーション全部盛り禁止
6. **比較表は使わない or 正直に書く** — 匿名 A 社 B 社との全勝比較禁止。弱みも認める
7. **感情は「シーン」で描く** — チェックボックス羅列禁止。読者が「自分のことだ」と感じる場面描写
8. **実物を見せる** — 実際の画面スクショ、講師写真等。モック禁止
9. **CTA は最大2箇所** — ヒーロー直後 + 最下部。4〜5箇所の連打禁止
10. **「引き」の余白を作る** — 3セクション続いたら余白セクションを挟む

### NGパターン（これが出たら AI 臭い → 即修正）
- 「たった1日で」「誰でも」「簡単に」「すぐに」→ 条件を明示
- □ 〜な方 □ 〜な方 のチェックボックス羅列 → シーン描写に変更
- 「こんなお悩みありませんか？」→ 具体シーンに置換
- 根拠なし数字（満足度98%等）→ ソースつけるか消す
- 全セクション同じ背景グラデーション → セクションごとに変化をつける

### コピー
- ヘッドラインはターゲットの最大の壁を否定 + ベネフィット（具体的な数値・期間入り）
- Problem は4つまで（チェックボックスではなくシーン描写で）
- 価格は割引・希少性・含まれるものリストを組み合わせる
- 「です・ます」だが固くしない。一文40文字以内。「！」は1ページ3回まで
- 「正直に言うと」等の本音トーンを混ぜる

### 演出
- オファーに合ったビジュアル演出を**1つだけ**入れる（1LP1手法）
- スクロールアニメーション（reveal）で自分のペースで読ませる
- glow や pulse は目立たせたい要素（CTA 等）だけ。全部光らせない
- CSS のみで演出できるならそうする

### やらない判断
- 素材がない写真・人物画像は入れない
- 架空のお客様の声は入れない
- 情報がないセクションは無理に入れない
- 1LP = 1オファーが基本。複数プランで迷わせない
- フリー素材の「笑顔のビジネスパーソン」写真、AI 生成人物イラスト禁止
