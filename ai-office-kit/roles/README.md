# roles/ — 職種別 役割テンプレ図書館

LINE AIオフィスの「社員の中身（役割層）」テンプレート集。
各ファイルは **役割層のみ**を持つ（共通層＝push型起動の動き方・リーダーへの集約報告ルール等は
`install.sh` が別途全社員に注入するため、ここには含めない）。

## 収録テンプレ（v1：8職種）

| 区分 | ファイル | 職種 |
|---|---|---|
| 🔧 コア必須 | `leader.md` | リーダー（統括役） |
| ⭐ プレミアム | `lp.md` | LP 担当（AI感を消す10原則） |
| ⭐ プレミアム | `designer.md` | デザイナー（多モデル並列生成） |
| ⭐ プレミアム | `writer.md` | ライター（セールス8ステップ） |
| ⭐ プレミアム | `video.md` | 動画制作担当（字幕焼き込み設定） |
| ⭐ プレミアム | `sns.md` | SNS 担当（ヒアリング＋文字数規律） |
| ⭐ プレミアム | `researcher.md` | リサーチャー（一次情報必須型） |
| ⚪ 汎用クリーン | `analyst.md` | アナリスト（数値分析） |

> 当面除外：product（自社テンプレ依存が強く、汎用化は後フェーズ）
> 商品から除外：brunson（書籍複製の著作権リスクのため製品には含めない）

## 使い方（install.sh が組み立てる）

`MEMBERS` の書式：`"<dir>:<表示名>:<role-template.md>"`

例：
```
MEMBERS=(
  "member-leader:リーダー:leader.md"
  "member-lp:LP担当:lp.md"
  "member-designer:デザイナー:designer.md"
)
```

`install.sh` は各社員について：
1. 指定の role-template を `members/<dir>/CLAUDE.md` へ配置
2. 共通層（session-mode テンプレ）を追記

の順で組み立てる。

## カスタマイズ
- 図書館に無い顧客独自職種は、代行セットアップ時に1枚書き起こす（主要な課金ポイント）。
- 各テンプレ内の固有名詞・案件名は導入時に顧客に合わせて調整する。
