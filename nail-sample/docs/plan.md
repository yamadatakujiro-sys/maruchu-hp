# 架空ネイルサロンHP制作計画（Figma活用フロー）

## Context

nail-walea.jp（表参道・外苑前のネイルサロンwalea）のような、上質な大人の女性向け**架空ネイルサロンのHP**を制作する。前回の歯科サンプルと異なり、今回は本格的な制作フローを踏む：**参考サイトの構造を完璧に読み取る → Figmaでデザインカンプを作成 → ユーザー承認 → コーディング**。

制作範囲は **TOP + 主要下層ページ**（メニュー／スタッフ／ご予約・アクセス を想定）。

### 確認済みの前提・制約

- **ネットワーク制限**: 現環境では nail-walea.jp が許可リスト外（403/ブロック確認済み）。ユーザーが環境設定で許可リストに追加する（→ステップ0）
- **Figma権限**: 接続済みだが現在 **Viewシート（Starterプラン）**。ファイル作成・編集が失敗する可能性があり、実行時に検証する。不可の場合はユーザーに編集権限への変更を依頼、それも難しければ「HTMLで直接デザイン→画像で承認」にフォールバック
- **著作権への配慮**: 参考サイトはあくまで「構成・情報設計・雰囲気の参考」。文章・サロン名・ロゴ・写真はすべてオリジナル（架空）とし、デザインの丸写しはしない
- **検索で判明している参考サイトの構成**: TOP／ネイル&ケアメニュー／ウェディングネイル／スタッフ紹介／ご予約からの流れ（FLOW）／サロン案内&アクセス／新着情報／ブログ&SNS／予約・問い合わせ。コンセプトは「walea＝ハワイ語で安らぎ」

## 制作プロセス

### ステップ0：ネットワーク許可（ユーザー作業 + 動作確認）
1. ユーザーが claude.ai/code の環境設定（ネットワークアクセス）に `nail-walea.jp` を追加
2. 実行開始時に Playwright でアクセス確認（`/tmp/analyze-walea.cjs` を再利用、`ignoreHTTPSErrors: true` 必須）。まだブロックされる場合は新規セッションが必要な旨をユーザーに伝える

### ステップ1：参考サイトの完全分析
- PlaywrightでTOP+下層ページ（/menu/ /staff/ /flow/ /access/ /contact/）をPC幅1280px・スマホ幅375pxでフルページ撮影
- DOM解析で抽出：セクション構成、ナビ項目、見出し階層、配色（CSS変数・計算済みスタイル）、フォント、余白・グリッド、CTAの種類と配置
- 分析結果を `nail-sample/docs/site-analysis.md` にまとめる（情報設計の設計書を兼ねる）

### ステップ2：架空サロンのコンセプト設計
- サロン名・コンセプト・ロゴ（SVG）・カラーパレット・フォント選定を提案（例：ハワイ語つながりの架空名、上品なベージュ×ゴールド系など。分析結果を踏まえ決定）
- ページ構成案（サイトマップ）と各ページのワイヤー概要をユーザーに提示し、AskUserQuestionで方向性を確認

### ステップ3：Figmaデザインカンプ作成 → 承認
1. まず `mcp__Figma__create_new_file` でファイル作成を試行（**Viewシート権限の検証ポイント**）
2. `/figma-use` スキルを読み込んでから `use_figma` でカンプ作成。順序：TOPページ（モバイル375px → PC1440px）→ 承認 → 下層ページ → 承認
3. `get_screenshot` でカンプ画像をユーザーに送付し、FigmaのURLも共有。修正指示を反映するループ
4. **権限エラー時のフォールバック**: ユーザーへ編集シート化を依頼、不可ならHTMLプロトタイプ直接制作（歯科サンプル方式）に切り替え

### ステップ4：コーディング
- `nail-sample/` フォルダに静的サイトとして実装（既存の居酒屋HP・dental-sampleには触れない）
  - `index.html` / `menu.html` / `staff.html` / `access.html`（予約・アクセス・流れを統合）
  - 共通 `style.css`（dental-sample同様CSS変数管理・日本語コメント・モバイルファースト）
- `get_design_context` で承認済みFigmaカンプを読み取り、忠実に実装
- 写真は当面プレースホルダ（CSS装飾/SVG）。実写真はユーザー提供 or 後日差し替え可能な構造にする

### ステップ5：検証・納品
- `python3 -m http.server` + Playwright で全ページ×3サイズ（375/768/1280）を撮影しユーザーに送付（フルページではなくセクション分割で大きく見せる）
- ページ間ナビゲーションのリンク切れ確認
- `claude/upbeat-meitner-v7va1g` ブランチにコミット・push（PRは作らない）

## 再利用するもの

- `/tmp/analyze-walea.cjs`（サイト分析スクリプト、許可後に再実行）
- `/tmp/shot2.cjs`（セクション別スクリーンショット撮影スクリプト）
- `dental-sample/style.css` のCSS設計パターン（変数定義・ブレークポイント768/1024・固定CTAバー）

## マイルストーン（承認ポイント）

2. TOPカンプ承認（ステップ3）
3. 下層カンプ承認（ステップ3）
4. 実装後の最終確認（ステップ5）
